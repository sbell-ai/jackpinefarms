Implement three enhancements to Product creation/editing (admin):

A) WYSIWYG product description (Tiptap)

1) In the admin Product create/edit form, replace the plain textarea for description with a Tiptap rich text editor (React).

2) Store the description as a single canonical format:

- Save to DB as HTML (preferred for simplest rendering) OR as JSON (Tiptap document). Pick one and use it everywhere.

3) Add Zod validation:

- description is required (or optional if currently optional) and has a max length safeguard.

4) On the product detail page and product listing cards, render the same stored content safely:

- If HTML: sanitize on the server before saving (or sanitize on render) to avoid script injection.
- If JSON: render via Tiptap read-only renderer.

B) Up to 5 product images (Cloudflare R2) + ordering

1) Update the database schema to support up to 5 images per product, with a stable order.

- Preferred: new `product_images` table: `id`, `product_id` FK, `r2_key`, `url`, `sort_order` (1–5), `alt_text` optional, `created_at`.
- Enforce max 5 images per product at the API level (and ideally via a constraint/guard).

2) Backend API:

- Add endpoints to upload/add/remove/reorder product images (admin-only).
- Upload flow: accept file -> validate type (jpeg/png/webp) + size limit -> upload to R2 via S3 SDK -> store row in `product_images`.

3) Admin UI:

- Allow selecting up to 5 images.
- Show thumbnails, allow reordering (drag/drop or up/down buttons), allow delete.
- Show clear errors when user tries to exceed 5 or uploads invalid files.

C) Same images on desktop + mobile (single source of truth + consistent rendering)

1) Remove any separate “mobile image” vs “desktop image” fields/logic for products.

2) Product pages must render from the single `product_images` array in a consistent way across breakpoints:

- Use one responsive component that always uses the same image set.
- Use CSS to adjust layout only (grid/stack), not which image is chosen.

3) If using `<picture>`/`srcSet`, it must be the same underlying image(s) (only different sizes), not different assets for mobile vs desktop.

Acceptance criteria

- Product form supports rich text editing and persists it correctly.
- Admin can upload 1–5 images, reorder them, and delete them; API enforces max 5.
- Product pages show the exact same images on desktop and mobile (only layout changes).
- All changes are TypeScript-safe, validated with Zod, and use existing auth/admin guards.
- Images are stored in Cloudflare R2 (no local disk persistence).
