Add Admin-only creation of Customers + Orders by extending the existing storefront checkout order system (do not create a parallel order model).

1) Customers (new DB primitive)

- Add a `customers` table (Drizzle + Postgres) and wire it into the existing Orders table via `orders.customer_id`.
- Minimal fields: `id (serial)`, `full_name (required)`, `email (optional)`, `phone (optional)`, `notes (optional)`, `created_at`.
- Add Zod insert schema + types.

2) Extend existing Orders schema (do not duplicate)

- Add `customer_id` FK (nullable for guest storefront orders, required for admin-created orders).
- Add `created_by` enum or nullable fields to identify source:
    - `source = "storefront" | "admin"`
    - (optional) `created_by_admin_user_id`
- Ensure `order_items` already store a snapshot price (`unit_price_cents`) and `quantity`; if not, add it so admin orders are stable even if product prices change later.

3) Admin API routes (spec-first + codegen)

Add admin-only endpoints (all behind `requireAdmin`) that reuse the same order creation logic used by storefront checkout:

- Customers:
    - `GET /admin/customers` (search)
    - `POST /admin/customers`
- Orders:
    - `POST /admin/orders` create a `draft` order with `customer_id` and `source="admin"`
    - `POST /admin/orders/{orderId}/items` add items (productId + quantity) and compute line totals using the same pricing rules as checkout (including egg dozen rule)
    - `POST /admin/orders/{orderId}/finalize` locks totals and marks status `placed` (and either `unpaid` for cash-at-pickup or creates a Stripe Checkout Session link)
    - `GET /admin/orders/{orderId}` returns order + customer + items + totals
- Do not bypass existing validation; reuse shared pricing/totals code where possible.

4) Admin UI (Shadcn + RHF + Zod + TanStack Query)

- Add `/admin/customers` page: create + search/list customers.
- Add `/admin/orders/new` page:
    - select existing customer or create inline
    - add products + quantities
    - show totals
    - choose payment method:
        - Cash at pickup (no Stripe required)
        - “Generate Stripe payment link” (Checkout Session URL saved to order and shown in UI)
- Ensure admin-created orders appear in the existing Admin Orders list and order detail page.

5) Stripe behavior (keep customer experience consistent)

- If generating a payment link:
    - Create a Stripe Checkout Session using the same line item logic as storefront checkout.
    - Store `stripe_checkout_session_id` + `checkout_url` on the order.
- No duplicate Stripe products/prices beyond what your existing system already uses.

Definition of Done

- Admin can create a customer.
- Admin can create an order, add items, finalize, and see it in the same Orders list as storefront orders.
- Totals match storefront checkout math (including eggs sold per dozen quantity semantics).
- TypeScript + Zod validation present; no parallel order tables created.
